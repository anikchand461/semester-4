# 🌳 ID3 vs C4.5 — Decision Tree Algorithms Compared

> Both **ID3** and **C4.5** build Decision Trees using Entropy — but they differ in **how they score a feature's usefulness**.
> We'll use the exact same 15-row dataset and compute everything step by step.

# 🅰️ METHOD 1 — ID3 Algorithm

## 📖 What is ID3?

**ID3 (Iterative Dichotomiser 3)** was introduced by **Ross Quinlan (1986)**.

It selects the best feature to split on using **Information Gain (IG)**:

```
IG(S, F) = H(S)  -  Σ (|Sᵥ|/|S|) · H(Sᵥ)
```

The feature with the **highest IG** becomes the split node.

### ⚠️ ID3 Weakness

ID3 **favours features with many unique values** — a feature like "Row ID" (15 unique values) would get artificially high IG even though it's useless. C4.5 was built to fix this.

---

## 📐 ID3 — Full Calculations

### Feature: Outlook (3 values)

| Outlook  | Count | Yes | No  |
| -------- | ----- | --- | --- |
| Sunny    | 6     | 3   | 3   |
| Overcast | 4     | 4   | 0   |
| Rain     | 5     | 3   | 2   |

```
H(Sunny)    = -(3/6·log₂3/6) - (3/6·log₂3/6) = 1.0000
H(Overcast) = -(4/4·log₂4/4) - 0              = 0.0000
H(Rain)     = -(3/5·log₂3/5) - (2/5·log₂2/5) = 0.9710

Weighted H  = (6/15)·1.0 + (4/15)·0.0 + (5/15)·0.971
            = 0.4000 + 0.0000 + 0.3237 = 0.7237

IG(Outlook) = 0.9710 - 0.7237 = 0.2473
```

---

### Feature: Temperature (3 values)

| Temperature | Count | Yes | No  |
| ----------- | ----- | --- | --- |
| Hot         | 5     | 3   | 2   |
| Mild        | 6     | 4   | 2   |
| Cool        | 4     | 3   | 1   |

```
H(Hot)  = -(3/5·log₂3/5) - (2/5·log₂2/5) = 0.9710
H(Mild) = -(4/6·log₂4/6) - (2/6·log₂2/6) = 0.9183
H(Cool) = -(3/4·log₂3/4) - (1/4·log₂1/4) = 0.8113

Weighted H  = (5/15)·0.971 + (6/15)·0.918 + (4/15)·0.811
            = 0.3237 + 0.3673 + 0.2163 = 0.9073

IG(Temperature) = 0.9710 - 0.9073 = 0.0637
```

---

### Feature: Humidity (2 values)

| Humidity | Count | Yes | No  |
| -------- | ----- | --- | --- |
| High     | 7     | 3   | 4   |
| Normal   | 8     | 6   | 2   |

```
H(High)   = -(3/7·log₂3/7) - (4/7·log₂4/7) = 0.9852
H(Normal) = -(6/8·log₂6/8) - (2/8·log₂2/8) = 0.8113

Weighted H  = (7/15)·0.9852 + (8/15)·0.8113
            = 0.4597 + 0.4327 = 0.8924

IG(Humidity) = 0.9710 - 0.8924 = 0.0786
```

---

### Feature: Wind (2 values)

| Wind   | Count | Yes | No  |
| ------ | ----- | --- | --- |
| Weak   | 9     | 6   | 3   |
| Strong | 6     | 3   | 3   |

```
H(Weak)   = -(6/9·log₂6/9) - (3/9·log₂3/9) = 0.9183
H(Strong) = -(3/6·log₂3/6) - (3/6·log₂3/6) = 1.0000

Weighted H  = (9/15)·0.9183 + (6/15)·1.0000
            = 0.5510 + 0.4000 = 0.9510

IG(Wind) = 0.9710 - 0.9510 = 0.0200
```

> ✅ **ID3 picks → Outlook** as the Root Node (highest IG = 0.2473)

# 🅱️ METHOD 2 — C4.5 Algorithm

## 📖 What is C4.5?

**C4.5** was also introduced by **Ross Quinlan (1993)** as a direct improvement over ID3.

Instead of raw Information Gain, C4.5 uses **Gain Ratio** — which **penalises features that have too many unique values**:

```
                    IG(S, F)
Gain Ratio(S, F) = ──────────────
                   SplitInfo(S, F)
```

### Split Info — the Penalty Term

```
SplitInfo(S, F) = -Σ (|Sᵥ|/|S|) · log₂(|Sᵥ|/|S|)
```

SplitInfo is itself an entropy — but measured over the **proportions of the split**, not the class labels. A feature that splits data into many tiny groups gets a **large SplitInfo**, which shrinks its Gain Ratio.

---

## 📐 C4.5 — Full Calculations

### Feature: Outlook

```
Split proportions: Sunny=6/15, Overcast=4/15, Rain=5/15

SplitInfo(Outlook) = -(6/15·log₂6/15) - (4/15·log₂4/15) - (5/15·log₂5/15)
                   = -(0.4·log₂0.4) - (0.267·log₂0.267) - (0.333·log₂0.333)
                   = -(0.4·-1.3219) - (0.267·-1.9069) - (0.333·-1.5849)
                   = 0.5288 + 0.5091 + 0.5278
                   = 1.5658

Gain Ratio(Outlook) = IG / SplitInfo
                    = 0.2473 / 1.5658
                    = 0.1579
```

---

### Feature: Temperature

```
Split proportions: Hot=5/15, Mild=6/15, Cool=4/15

SplitInfo(Temperature) = -(5/15·log₂5/15) - (6/15·log₂6/15) - (4/15·log₂4/15)
                       = -(0.333·-1.585) - (0.4·-1.322) - (0.267·-1.907)
                       = 0.5283 + 0.5288 + 0.5091
                       = 1.5662

Gain Ratio(Temperature) = 0.0637 / 1.5662
                        = 0.0407
```

---

### Feature: Humidity

```
Split proportions: High=7/15, Normal=8/15

SplitInfo(Humidity) = -(7/15·log₂7/15) - (8/15·log₂8/15)
                    = -(0.467·-1.099) - (0.533·-0.907)
                    = 0.5130 + 0.4834
                    = 0.9966

Gain Ratio(Humidity) = 0.0786 / 0.9966
                     = 0.0789
```

---

### Feature: Wind

```
Split proportions: Weak=9/15, Strong=6/15

SplitInfo(Wind) = -(9/15·log₂9/15) - (6/15·log₂6/15)
                = -(0.6·-0.7370) - (0.4·-1.3219)
                = 0.4422 + 0.5288
                = 0.9710

Gain Ratio(Wind) = 0.0200 / 0.9710
                 = 0.0206
```

---

| Feature     | IG (from ID3) | SplitInfo | Gain Ratio | Rank   |
| ----------- | ------------- | --------- | ---------- | ------ |
| Outlook     | 0.2473        | 1.5658    | **0.1579** | 🥇 1st |
| Humidity    | 0.0786        | 0.9966    | 0.0789     | 🥈 2nd |
| Temperature | 0.0637        | 1.5662    | 0.0407     | 🥉 3rd |
| Wind        | 0.0200        | 0.9710    | 0.0206     | 4th    |

> ✅ **C4.5 also picks → Outlook** (highest Gain Ratio = 0.1579), but the **ranking logic is different**

# ⚖️ ID3 vs C4.5 — Head to Head

| Aspect              | ID3                               | C4.5                         |
| ------------------- | --------------------------------- | ---------------------------- |
| **Scoring metric**  | Information Gain                  | Gain Ratio                   |
| **Bias**            | Favours high-cardinality features | Corrects for cardinality     |
| **Penalty term**    | None                              | SplitInfo                    |
| **Missing values**  | Cannot handle                     | ✅ Can handle                |
| **Continuous data** | Cannot handle                     | ✅ Can handle (thresholding) |
| **Pruning**         | No pruning                        | ✅ Post-pruning support      |
| **Output**          | Tree only                         | Tree + rules                 |
| **Year**            | 1986                              | 1993                         |
| **Invented by**     | Ross Quinlan                      | Ross Quinlan                 |

# 💻 Full Python Code — Both Methods

```python
import numpy as np
import pandas as pd
from math import log2
from collections import Counter

# ── Dataset ──────────────────────────────────────────────────────────────────
data = {
    'Outlook':     ['Sunny','Sunny','Overcast','Rain','Rain','Rain','Overcast',
                    'Sunny','Sunny','Rain','Sunny','Overcast','Overcast','Rain','Sunny'],
    'Temperature': ['Hot','Hot','Hot','Mild','Cool','Cool','Cool',
                    'Mild','Cool','Mild','Mild','Mild','Hot','Mild','Hot'],
    'Humidity':    ['High','High','High','High','Normal','Normal','Normal',
                    'High','Normal','Normal','Normal','High','Normal','High','Normal'],
    'Wind':        ['Weak','Strong','Weak','Weak','Weak','Strong','Strong',
                    'Weak','Weak','Weak','Strong','Strong','Weak','Strong','Weak'],
    'PlayTennis':  ['No','No','Yes','Yes','Yes','No','Yes',
                    'No','Yes','Yes','Yes','Yes','Yes','No','Yes']
}
df = pd.DataFrame(data)
TARGET   = 'PlayTennis'
FEATURES = ['Outlook', 'Temperature', 'Humidity', 'Wind']

# ── Core Functions ────────────────────────────────────────────────────────────
def entropy(labels):
    """Shannon entropy of a label series."""
    counts = Counter(labels)
    total  = len(labels)
    return -sum((c/total) * log2(c/total) for c in counts.values() if c > 0)

def split_info(df, feature):
    """SplitInfo penalty used by C4.5."""
    total  = len(df)
    result = 0
    for val in df[feature].unique():
        prop    = len(df[df[feature] == val]) / total
        result -= prop * log2(prop) if prop > 0 else 0
    return result

def information_gain(df, feature, target):
    """ID3 metric — raw Information Gain."""
    total   = len(df)
    base_h  = entropy(df[target])
    w_entropy = sum(
        (len(sub := df[df[feature] == v][target]) / total) * entropy(sub)
        for v in df[feature].unique()
    )
    return base_h - w_entropy

def gain_ratio(df, feature, target):
    """C4.5 metric — Gain Ratio = IG / SplitInfo."""
    ig = information_gain(df, feature, target)
    si = split_info(df, feature)
    return ig / si if si != 0 else 0

# ── Compute & Display ─────────────────────────────────────────────────────────
base_entropy = entropy(df[TARGET])

print("=" * 65)
print(f"  Base Entropy H(S) = {base_entropy:.4f}")
print("=" * 65)

# ── ID3 Table ─────────────────────────────────────────────────────────────────
print("\n📊 ID3 — Information Gain")
print(f"{'Feature':<15} {'SplitInfo':>10} {'IG':>10}  {'Rank'}")
print("-" * 50)

ig_scores = {f: information_gain(df, f, TARGET) for f in FEATURES}
for rank, (feat, ig) in enumerate(sorted(ig_scores.items(),
                                         key=lambda x: -x[1]), 1):
    si = split_info(df, feat)
    medal = ["🥇","🥈","🥉","  "][rank-1]
    print(f"{feat:<15} {si:>10.4f} {ig:>10.4f}  {medal} #{rank}")

best_id3 = max(ig_scores, key=ig_scores.get)
print(f"\n  ✅ ID3  Root Node → {best_id3}  (IG = {ig_scores[best_id3]:.4f})\n")

# ── C4.5 Table ────────────────────────────────────────────────────────────────
print("📊 C4.5 — Gain Ratio")
print(f"{'Feature':<15} {'IG':>8} {'SplitInfo':>12} {'GainRatio':>12}  {'Rank'}")
print("-" * 60)

gr_scores = {f: gain_ratio(df, f, TARGET) for f in FEATURES}
for rank, (feat, gr) in enumerate(sorted(gr_scores.items(),
                                          key=lambda x: -x[1]), 1):
    ig = ig_scores[feat]
    si = split_info(df, feat)
    medal = ["🥇","🥈","🥉","  "][rank-1]
    print(f"{feat:<15} {ig:>8.4f} {si:>12.4f} {gr:>12.4f}  {medal} #{rank}")

best_c45 = max(gr_scores, key=gr_scores.get)
print(f"\n  ✅ C4.5 Root Node → {best_c45}  (GR = {gr_scores[best_c45]:.4f})\n")

# ── Agreement check ───────────────────────────────────────────────────────────
print("=" * 65)
if best_id3 == best_c45:
    print(f"  🤝 Both ID3 and C4.5 agree: '{best_id3}' is the Root Node!")
else:
    print(f"  ⚠️  ID3 picks '{best_id3}' but C4.5 picks '{best_c45}'")
print("=" * 65)
```

# 🔍 Key Insight — Why Rankings Differ

Notice how **Temperature and Wind swap ranks** between ID3 and C4.5:

```
             ID3 Rank   C4.5 Rank   Why?
Temperature    3rd  →     4th      Has 3 values → high SplitInfo (1.566)
                                   → Gain Ratio gets penalised heavily
Wind           4th  →     3rd      Has only 2 values → low SplitInfo (0.971)
                                   → Penalty is smaller → Gain Ratio rises
```

This is exactly the **bias that C4.5 was designed to fix** — it stops the tree from blindly favouring features with more categories.

# 📝 Formula Cheat Sheet

```
──────────────────────────────────────────────────────────────
  SHARED
  Entropy:     H(S) = -Σ pᵢ · log₂(pᵢ)

──────────────────────────────────────────────────────────────
  ID3
  Info Gain:   IG(S,F) = H(S) - Σ(|Sv|/|S|)·H(Sv)
  Best split:  argmax_F [ IG(S, F) ]

──────────────────────────────────────────────────────────────
  C4.5
  SplitInfo:   SI(S,F) = -Σ (|Sv|/|S|) · log₂(|Sv|/|S|)
  Gain Ratio:  GR(S,F) = IG(S,F) / SI(S,F)
  Best split:  argmax_F [ GR(S, F) ]

──────────────────────────────────────────────────────────────
```

---

> 💡 **Rule of thumb:** Use **ID3** to understand the concept. Use **C4.5** (or its commercial successor **C5.0**) for real datasets with many-valued categorical features or continuous variables.
